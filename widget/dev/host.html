OyeChats Widget — Host Fixture

Mimics a customer page. Use to verify the widget loads, the bundle splits correctly, and the public API works.

Verify the bundle split (Phase 1). Open DevTools → Network. You should see:
  • oyechats-widget.js — the loader, ≤ 8 KB gzipped, loaded by the script tag
  • manifest.json + oyechats-app.[hash].js + oyechats-vendor.[hash].js — loaded immediately after
  • oyechats-chat.[hash].js — appears only on first widget open
  • oyechats-livechat.[hash].js — appears only on "talk to human" handoff
  • oyechats-forms.[hash].js — appears only when a form (lead capture, handoff, booking) is shown
Test the public API (Phase 3).
Test deferred init (Phase 2). To test, open the browser console and run: 
window.OYECHATS_ASYNC_INIT = true; location.reload();
// Then later:
OyeChats.init();

Embed (matches what customers paste into their site):

<script src="/oyechats-widget.js" data-bot-key="bot-DEV"></script>
